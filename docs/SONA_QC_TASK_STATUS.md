# Статус задачи: форма для Соны, отчёт по КК и уведомления в Telegram

**Задача:** [КК бух.услуг] Создать форму для Соны, отчёт по КК и уведомление в Telegram.
**Дата оценки:** 2026-07-20 · Репозиторий: `kk-accountants-feedback-form` (ветка `main`).

Ниже — честная сверка каждого пункта задачи с тем, что реально есть в коде этого
приложения. Важный контекст архитектуры: **Сона заполняет свои формы на своей
QA-платформе** (`sona-qa-platform`, таблицы `sqa_tickets` / `sqa_reviews`) — точно
так же, как Маргарита работает в `mqa_*`. Это приложение **не** является местом ввода
форм: оно забирает результаты Соны через `kk_ingest_problems()` (миграция 0021) и
показывает их бухгалтерам/руководству. Поэтому часть пунктов задачи относится к
платформе Соны (репозиторий #1), а не к этому репозиторию.

## Что уже сделано (готово)

- **Приёмка результатов Соны в тикеты** — `kk_ingest_problems()` (миграция 0021, `src/lib/ingestion.js`):
  `sqa_tickets` и `sqa_reviews` (`record_type='problem'`) → `kk_problems` с source `sona_review`,
  статус `waiting_for_accountant`. Ключи стабильные (`sona_review:<id>`), upsert идемпотентный.
- **Скрытие личности Соны** (0018) — источник показывается нейтрально «Качество бухгалтерской работы».
- **Приоритеты тикетов** — `priority` 1–3, SLA в рабочих часах Маргариты (`dashboard.js`).
- **Реакция бухгалтера на тикет** — «Ознакомлен» / «Подать апелляцию», штрафы, разбор апелляций (0025–0027).
- **Дашборд по бухгалтерам** с фильтрами периода (Сегодня / 2 дня / Неделя / Всё время), только Маргарита+Сона, без AI (0023).

## Чего не хватает — что нужно сделать

1. **Форма ввода Соны** (репозиторий #1, `sona-qa-platform`): вкладка с выбором компании,
   автоподтягиванием бухгалтера и связанных форм, ежедневный ввод — по образцу формы Маргариты.
   В этом репозитории — только приёмка; саму форму надо согласовать и делать на платформе Соны.
2. **Тип записи: проблема / похвала / другое.** Ввести положительные оценки: если всё хорошо —
   не тикет, а «похвала». Нужны: поле типа в источнике, приёмка положительных оценок в
   `kk_problems`/новую сущность, отражение на дашборде и в отчётах. Сейчас сохраняются только проблемы.
3. **Отметка «ОЧЕНЬ СРОЧНО»** — отдельный уровень/визуальный флаг сверх приоритета 1
   (например, `priority=0` или булев `urgent`) с выделением в очереди бухгалтера.
4. **Отчёт по работе Соны** — аналог отчёта Маргариты (`reports.js`/`Reports.jsx`):
   сколько проверено в день, оценки, разбивка по бухгалтерам, эффективность. Нужен источник
   «проверено чатов/компаний Соной» (аналог `kk_margarita_checks`).
5. **Сводный отчёт по качеству отдела и по каждому бухгалтеру** за день/неделю/месяц,
   объединяющий данные Соны и Маргариты в **один** ежедневный отчёт.
6. **Автоматическая рассылка отчётов в Telegram (ОК)** — ежедневно и еженедельно.
   Нужны: Supabase Edge Function (или сервис) с токеном бота (service_role — вне фронтенда и git),
   расписание (pg_cron), формирование текста отчёта. Сейчас нет ничего из этого.
7. **Опрос Соны «что улучшить в платформе»** после 3-й проверки / по вечерам.
8. **Организационные шаги:** согласование формы с Лилит/Эмилией/Соной, проверка 10 форм,
   сбор обратной связи и сверка отчёта после внедрения.

## Примечание по архитектуре

Формы Соны и Маргариты — это **их QA-платформы** (репозиторий #1). Это приложение —
«приёмник» (см. `docs/INTEGRATION_PROMPT.md`): оно забирает результаты через
`kk_ingest_problems()`, показывает тикеты бухгалтерам и строит отчёты для руководства.
Поэтому пункты «форма Соны» и «тип похвала/оценки» требуют изменений **на платформе Соны**
(источник данных) + приёмки и отображения здесь; а «отчёт по работе Соны», «сводный отчёт
по отделу» и «Telegram-рассылка» реализуются в этом репозитории.
