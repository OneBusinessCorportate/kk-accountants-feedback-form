# Статус задачи: форма для Соны, отчёт по КК и уведомления в Telegram

**Задача:** [КК бух.услуг] Создать форму для Соны, отчёт по КК и уведомление в Telegram.
**Обновлено:** 2026-07-20 · Репозиторий: `kk-accountants-feedback-form` (ветка `main`).

Контекст архитектуры: **Сона заполняет свои формы на своей QA-платформе**
(`sona-qa-platform`, таблицы `sqa_reviews` / `sqa_tickets`), как Маргарита в
`mqa_*`. Это приложение — «приёмник»: оно забирает результаты и показывает
тикеты/отчёты. Поэтому пункт «форма ввода Соны» относится к репозиторию #1, а
отчёты/похвала/Telegram — к этому репозиторию (реализованы здесь).

## Что сделано в этом релизе

**Миграции БД (применены к OB FAQ, проверены на реальных данных):**
- `0030_sona_report_and_praise.sql` — read-only view `kk_sona_checks` (175 проверок
  Соны, все с распознанным бухгалтером) + таблица `kk_praise` и функция
  `kk_ingest_praise()` (6 554 похвалы: 6 408 оценок Маргариты «Хорошо»/«Отлично» +
  146 чистых проверок Соны), с cron каждые 30 мин. Аддитивно — `kk_ingest_problems()`
  не тронута.
- `0031_schedule_quality_report_telegram.sql` — cron (pg_cron + pg_net) на ежедневную
  (Пн–Пт 19:30) и еженедельную (Пн 09:30, Ереван) отправку отчёта. Безопасна:
  ничего не планирует, пока не заданы `app.edge_base_url` / `app.edge_auth`.

**Edge Function:** `supabase/functions/quality-report-telegram/` — строит отчёт по
отделу + бухгалтерам и шлёт в Telegram ОК (`?period=daily|weekly`).

**Логика (чистые модули + тесты):**
- `dashboard.js` — `urgencyLevel()` / `isVeryUrgent()` + категория `urgent`.
- `reports.js` — `buildSonaReport()`, `summarizePraise()`, `praiseByAccountant()`.
- `qualityReport.js` — `buildQualityReport()` (отдел + по бухгалтерам), `urgentIssues()`.
- `telegramReport.js` — форматирование сообщения (спек для Edge Function).
- `constants.js` — `URGENCY*`, `FEEDBACK_TYPE*`.
- `api.js` — `fetchSonaChecks()`, `fetchPraise()`.

**UI:** дашборд (карточка «ОЧЕНЬ СРОЧНО», стат «Похвалы», бейдж срочности),
карточка бухгалтера (бейдж срочности), `/reports` (вкладки «Отдел», «Работа Соны»
+ предпросмотр Telegram).

Тесты: 242 зелёных; сборка проходит.

1. **Развернуть Telegram-бота** (пункт 8): `supabase functions deploy
   quality-report-telegram`, задать секреты `TELEGRAM_BOT_TOKEN` / `TELEGRAM_CHAT_ID`,
   затем настройки `app.edge_base_url` / `app.edge_auth` и применить `0031`.
   Инструкция — `docs/TELEGRAM_REPORT_SETUP.md`.
2. **Форма ввода Соны и опрос «что улучшить»** (пункты 1, 13) — на QA-платформе Соны
   (репозиторий #1).
3. **Организационное** (пункты 3, 14, 15): проверить 10 форм, согласовать с
   Лилит/Эмилией/Соной, собрать обратную связь и сверить отчёт после внедрения.
